# Implementation Plan: V2 Storefront Upgrade

**Branch**: `002-v2-storefront-upgrade` | **Date**: 2026-07-15 | **Spec**: [spec.md](file:///D:/مشاريع/Bayt%20Al-Ezz/specs/002-v2-storefront-upgrade/spec.md)

**Input**: Feature specification from `/specs/002-v2-storefront-upgrade/spec.md`

## Summary

Upgrade the Bayt Al-Ezz storefront to v2.0 with four major changes: (1) auto-opening house animation replacing click-to-open door, (2) admin visual icon picker for section management, (3) optional product images, and (4) comprehensive security hardening including Magic Link auth, RLS, rate limiting, and server-side validation. This will be implemented using a vanilla JS/HTML/CSS approach (no build step) with Supabase as the backend, deploying statically on Vercel.

## Technical Context

**Language/Version**: HTML5, CSS3, ES6+ JavaScript (no bundler/transpiler, plain browser modules)

**Primary Dependencies**: Supabase JS Client v2 (loaded via CDN/static script), Cairo Font (Google Fonts)

**Storage**: PostgreSQL (Supabase database) with RLS enabled; Supabase Storage for assets

**Testing**: Plain JS unit tests (assert scripts / lightweight testing) inside `/tests`

**Target Platform**: Modern mobile and desktop web browsers (RTL / Arabic primary)

**Project Type**: Static web application (no-build-step architecture)

**Performance Goals**:
- Page weight under 500KB
- House animation completes within 3s (including DB fetch)
- Client-side visual updates reflected under 30s
- Smooth transitions at 60 fps on mobile

**Constraints**:
- Must only use design token palette (`--primary-blue`, `--secondary-navy`, `--tertiary-gray`, `--neutral-gray`, `--base-white`)
- Arabic-first, RTL layout
- Mobile-first responsiveness
- Cairo typography
- No-build-step deployment on Vercel

**Unknowns**:
- Magic Link Redirect Flow in vanilla JS client: How to securely handle redirect tokens and route to the dashboard on login in a serverless setup? -> **NEEDS CLARIFICATION**
- SVG Frame 1 & Frame 2 structure and transition method: Can SVGs be dynamically injected or nested inline to enable CSS styling, coordinate-based HTML text overlays, and Tab navigability? -> **NEEDS CLARIFICATION**
- Client-side throttling and server-side validation constraints: What are the exact db constraints or RLS policies needed to reject invalid input, and how will throttling be implemented? -> **NEEDS CLARIFICATION**
